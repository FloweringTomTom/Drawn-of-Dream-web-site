Installation instructions for Wreathe 7.3: Elegiac

Page location: Software Wreathe Wreathe 7.3: Elegiac

Table of Contents

  1. Provisional system requirements
  2. Development Version Installation (not supported; for development and/or testing)
  3. Updating the development version
  4. Development instructions

Wreathe 7.3 is currently in development and is not yet supported. To install Wreathe, using a current version instead is recommended.

Provisional system requirements

  • Intel Core 2 CPU or compatible, with at least two cores
  • At least 64GB available disk space
  • Compatible GPU (Intel recommended)
  • Color LCD display
  • At least 8GB RAM
  • Compatible keyboard (any standard IBM-compatible should work, such as the IBM SpaceSaver II)
  • Compatible pointing tool (TrackPoint, mouse, etc.)

It is probably possible to install Wreathe 7.3 on computers that do not meet the recommended system requirements, but that is not supported.

Development Version Installation (not supported; for development and/or testing)

During development, Wreathe 7.3: Elegiac is distributed as a Portage overlay, and requires a working Gentoo GNU/Linux installation. To install it, first add the overlay to the system by running:

layman -o https://raw.githubusercontent.com/ethus3h/wreathe-overlay/master/wreathe-overlay.xml -f -a wreathe-overlay -p 100

Then, emerge the base package by running:
layman -s wreathe-overlay; emerge wreathe-base

Updating the development version

To update an installation of the development version, there are two techniques: First, you can update any changed live ebuilds on your system:

smart-live-rebuild

Alternatively, if you prefer, you can update only the Wreathe-specific components, by running:
layman -s wreathe-overlay; emerge wreathe-base wreathe-backgrounds wreathe-media-resources wreathe-office-resources wreathe-typefaces-redist wreathe-typeface-family compiz-extra-snowflake-textures

To update just the main package (this may break things), run:
layman -s wreathe-overlay; emerge wreathe-base

Development instructions

To save changes to the overlay (assuming you have cloned a fork locally to work on), run the following command in the git directory:

repoman manifest; git add *; git commit -am "Overlay update"; git push -u origin master
