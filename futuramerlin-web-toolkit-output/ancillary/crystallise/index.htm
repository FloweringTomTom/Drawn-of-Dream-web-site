Overview

An app to easily archive files stored in computer-accessible digital formats.

Documentation

Usage

Crystallise provides the following user-facing command-line scripts. See the individual pages for the various scripts for usage instructions.

  • crystallise
  • decrystallise
  • quickliquid
  • dequicklify
  • crystallise-update

In addition, it also provides several scripts that it uses internally.

Installation

For Wreathe 7.3 and similar operating systems (e.g. Gentoo GNU/Linux), an ebuild (app-misc/crystallise) is available in the Wreathe overlay.

For other operating systems, use the following installation instructions.

Download

To download Crystallise, run:

git clone https://github.com/ethus3h/crystallise.git

Requirements

  • UNIX-like operating system (Wreathe 7.3, Ubuntu, and Mac OS X 10.11 are currently supported.)
  • An account at the non-profit Internet Archive.
  • ia (https://pypi.python.org/pypi/internetarchive) 1.0.2 or later (must be configured by running ia configure since installing 1.0.2 or later before Crystallise can be used) (it is recommended to apply the Wreathe patches to internetarchive for using with crystallise).
  • a root login or administrator/sudo privileges for your local computer, both for installation and use of all tools except quickliquid and dequicklify.
  • wget 1.14 or later
  • git
  • hashdeep
  • (A recent version of bash)

Setup

Edit the configuration file; see the "Configuration file format" section for documentation of this.

Finally, run sudo make install.

Future plans

  • Move config.txt to something less generic.
  • Have decrystallise verify checksums.
  • Verify that the coal5 file can correctly be expanded to the original files before (or while?) uploading, and before deleting input files.
  • Add folder support to quickliquid.

Configuration file format

The configuration file is located in your system configuration directory (probably /usr/local/etc or /etc), and is named crystallise.conf. It is a comma-separated list containing configuration values separated by a line feed (0x0A), as follows:

  • Installation UUID
  • Internet Archive collection identifier (write access to the collection is required)
  • Passphrase (must be a valid GPG passphrase)
  • Directory for working data (should be writeable and have sufficient free space to hold approximately three times the amount of data being crystallised at any one time)

Note that there is currently no facility for storing configuration values containing line feeds.
