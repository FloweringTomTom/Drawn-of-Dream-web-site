The changes from upstream's ebuild in that one are:

* (mark all arches unstable)
* add "+bundled-heimdal" to IUSE
* replace "!system-mitkrb5? ( >=app-crypt/heimdal-1.5[-ssl,${MULTILIB_USEDEP}] )" with
     !system-mitkrb5? (
        !bundled-heimdal? (
            >=app-crypt/heimdal-1.5[-ssl,${MULTILIB_USEDEP}]
        )
    )
* Add winbind to addc's REQUIRED_USE
* Remove
    PATCHES=(
        "${FILESDIR}/${PN}-4.2.3-heimdal_compilefix.patch"
        "${FILESDIR}/${PN}-4.2.7-pam.patch"
    )
* Replace "epatch ${PATCHES[@]}" with
    PATCHES=(
        "${FILESDIR}/${PN}-4.2.7-pam.patch"
    )
    if ! use bundled-heimdal ; then
        PATCHES+="${FILESDIR}/${PN}-4.2.3-heimdal_compilefix.patch"
    fi
    epatch "${PATCHES[@]}"
* Delete --bundled-libraries=NONE" and "--builtin-libraries=NONE"
* Add before "if multilib_is_native_abi ; then":
    if use bundled-heimdal ; then
        myconf+=(
            "--builtin-libraries=heimdal"
        )
    else
        myconf+=(
            "--bundled-libraries=NONE"
            "--builtin-libraries=NONE"
        )
    fi
* Replace "waf-utils_src_configure ${myconf[@]}" with "waf-utils_src_configure "${myconf[@]}""

Corresponding changes to the unstable ebuilds should be possible as well, to resolve these issues.
