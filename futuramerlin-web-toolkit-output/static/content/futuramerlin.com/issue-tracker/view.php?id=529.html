Summary0000529: Window manager: Static App Switcher improvements
Description### Bugs

- When starting in reverse mode (shift-cmd-tab), releasing shift (but keeping cmd held) exits the switcher. This prevents navigating freely within the switcher by pressing or releasing shift to move forwards or backwards in the list. Navigating like that works fine when starting in forwards mode (cmd-tab).
- Selecting an app that has a window in the foreground that is _not_ included in the \\\\\\\\\\\\\\\"Switch windows\\\\\\\\\\\\\\\" match will not move the app to the front of the staticswitcher list as would be expected. I presume because the app\\\\\\\\\\\\\\\'s main window never gets activated.
- When exiting the switcher, the bottom-most window will flash briefly. This is tracked in [upstream issue 20](https://github.com/compiz-reloaded/compiz-plugins-main/issues/20). It can be prevented in some cases by applying [this patch](https://github.com/ethus3h/wreathe/blob/master/etc/portage/patches/x11-plugins/compiz-plugins-main/staticswitcher-fix-flickering-in-some-cases.patch), but not with some apps (e.g. the problem still occurs with the Atom text editor).
- A white border sometimes shows up briefly when opening the switcher.
- When using Emerald, the normal switcher decorations/border do not show up, or at least do not stay present during the time the switcher is open. This works fine when using gtk-window-decorator.
- The \\\\\\\\\\\\\\\"Minimized Window Highlight Rectangle\\\\\\\\\\\\\\\" should be drawn if it\\\\\\\\\\\\\\\'s enabled, even if \\\\\\\\\\\\\\\"Highlight Mode\\\\\\\\\\\\\\\" is set to \\\\\\\\\\\\\\\"Bring Selected To Front\\\\\\\\\\\\\\\" or \\\\\\\\\\\\\\\"None\\\\\\\\\\\\\\\".
- The border around the popup drawn by the window decorator is not blurred, even when blur is enabled (mentioned in [compiz issue 68](https://github.com/compiz-reloaded/compiz/issues/68)).

### Enhancements

- Option to display the title of the highlighted window, in the manner of the Shift Switcher
- Option to have the highlighted window follow the current mouse-hovered window thumbnail when the mouse is moved
- Ability to configure colors, shadow, and corner radius of the popup window
- Ability to configure color of the border drawn around the preview of the highlighted window\\\\\\\\\\\\\\\'s thumbnail
- Ability to cycle between windows from the same app ((shift)-cmd-backtick)
- Ability to animate opening and closing of the switcher popup
