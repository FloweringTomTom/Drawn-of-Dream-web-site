 Information known so far:

The issue is:

    Emerald crashes with the message emerald: /var/tmp/portage/x11-libs/cairo-1.14.6/work/cairo-1.14.6/src/cairo-scaled-font.c:459: _cairo_scaled_glyph_ page_destroy: Assertion!scaled_font->cache_frozen' failed.`

    gtk-window-decorator crashes with the message gtk-window-decorator: /var/tmp/portage/x11-libs/cairo-1.14.6/work/cairo-1.14.6/src/cairo-scaled-font.c:459: _cairo_scaled_glyph_page_destroy: Assertion!scaled_font->cache_frozen' failed.`.

The issue is not related to the selected Emerald theme.

The issue occurs both when Emerald is built with GTK2 and when it is built with GTK3.

The issue occurs with both pango-1.38.1 and 1.40.1.

The issue occurs with both en_GB.utf8 and en_US.utf8 locales (with a reboot after changing the locale).

The issue occurs when Noto Color Emoji is used for a character in a window title.

The issue does not affect Openbox (which does not seem to use Noto Color Emoji in window decorations at all, even for characters for which only it contains glyphs).

gtk-window-decorator works if there is a window title with a problem emoji in it (in any position) at the time when gtk-window-decorator is started.

However, if there is not a problem emoji in the title when it is started, it will crash when a window title is set to a string containing a problem emoji (in any position).

Emerald does not work if there is a window title with a problem emoji in it, crashing immediately if there is a problem emoji in a window title when it is started.

The position (start, middle, end) does not seem to affect the behaviour of either gtk-window-decorator or Emerald.

There is also another crash that needs its own issue, that occurs when using app switcher, but does not always happen.
