 Test results:

With a clean profile, scrolling on the tab pane without any addons installed works fine; not tested yet with Wreathe profile and Tree Style Tab (TST) disabled.

Scrolling on the tab pane with TST installed behaves as follows:

(GTK2) Firefox 45.3.0 ESR with Gentoo and Wreathe patches, Wreathe profile (my preferred setup): Switches tabs.
(GTK2) Firefox 45.3.0 ESR with Gentoo and Wreathe patches, clean profile: Glitchily switches tabs and scrolls (sometimes one, sometimes the other, sometimes both; glitches between them within <1 second).
(GTK2) Firefox 45.3.0 ESR official binary, Wreathe profile: Switches tabs.
(GTK2) Firefox 45.3.0 ESR official binary, clean profile: Glitchily switches tabs and scrolls (sometimes one, sometimes the other, sometimes both; glitches between them within <1 second).
((GTK3) Firefox 48.0 stable official binary not tested due to add-on signing requirement.)
(GTK3) Firefox 50.0a2 DevEd official binary, Wreathe profile: does nothing.
(GTK3) Firefox 50.0a2 DevEd official binary, clean profile: scrolls (expected behaviour).

To check: 45.3.0 ESR w/ patches, Wreathe profile, TST disabled; 48.0 stable official, clean profile; 45.3.0 ESR w/ patches, Wreathe profile, vanilla TST?

Short description

When using GTK2 builds of Firefox, scrolling using the mouse wheel on the tab panel switches through the tabs, rather than scrolling the panel.
Steps to reproduce

    Start Firefox 45.3.0 ESR, 64-bit GNU/Linux official binary, with clean profile.
    Install TST.
    Scroll tab panel using mouse wheel. ( 4. …profit?)

Expected result

Tab panel scrolls.
Actual result

A glitchy combination of scrolling the tab panel and switching between the various open tabs. (Scrolling slowly seems to make it more likely to scroll the panel, while scrolling quickly or past the end of the available selection of tabs seems to make it more likely to change which tab is selected.)
